# 存储芯片制造场景需求评审一页纸

## 一、评审对象

本次评审聚焦两个功能点：

1. 需求计划是否在现有产能约束下可执行
2. 当计划不可行时，系统给出的优化投片建议是否满足存储芯片制造场景

评审范围覆盖现有 `RCCP`、`LP 优化`、`Production Plan` 与 `Output 视角` 相关能力。

## 二、审查结论

1. 当前系统已具备基础的需求计划可执行性判断能力，但结论口径仍偏向“新计划静态负载”。
2. 对于存储芯片制造场景，当前主链路尚未将 WIP 后续工序负载完整纳入 RCCP / LP 约束，因此不能直接作为正式投片决策依据。
3. 不可行计划的优化投片功能必须考虑 WIP，且至少同时考虑两类影响：
   - WIP 对目标产出的贡献
   - WIP 对后续关键机台的持续占机负载

## 三、主要问题

### 1. 可执行性判断口径不完整

当前主判断逻辑主要基于：

`新需求计划小时 / 可用产能小时`

该口径没有完整纳入：

`WIP 后续工序小时 / 可用产能小时`

在存储厂长 CT、高重入场景下，可能出现“系统判断可行，现场执行过载”的偏差。

### 2. 优化投片建议存在失真风险

若系统仅根据产出缺口或新投片 TC 进行优化，会出现：

- 已有高完成度 WIP 时仍重复投片
- 现有 WIP 已占用瓶颈资源时仍建议继续投片
- 数学上可行、现场无法排产的优化结果

## 四、优化方案

采用“方案 A：最小闭环增强”，在现有框架上补齐 WIP 校正能力。

### 后端优化

1. `RCCP` 增加 `WIP-aware` 口径：
   - 总需求小时 = 新计划需求小时 + WIP 后续负载小时
2. `LP` 增加 WIP 基础占用约束：
   - `Σ C[i,j] * x[i] + WIP_base_load[j] <= H[j]`
3. `Production Plan` 输出双口径结果：
   - 原始计划结果
   - WIP 校正后结果
4. 复用 `Output RCCP` 中已有的 WIP 后续负载计算逻辑，避免重复建模。

### 前端优化

1. 在生产计划区域增加“WIP 校正模式”结果卡
2. 增加“原计划 vs WIP 校正后计划”对比表
3. 明确标识本次结论是否已考虑 WIP
4. 关联 Output 视角结果与 Production Plan，形成闭环展示

## 五、预期收益

1. 可执行性判断更贴近存储厂真实排产约束
2. 优化投片结果更适合正式业务决策
3. 评审、开发、验收三方口径统一，减少需求歧义

## 六、实施建议

1. 一期落地 `WIP-aware RCCP + WIP-aware LP + Production Plan 展示增强`
2. 前端同步展示“原计划结论”和“WIP 校正后结论”
3. 验证重点覆盖两类场景：
   - 无 WIP 时与现有逻辑保持兼容
   - 高 WIP 时能正确识别计划风险并收敛投片建议
